03 — Vulnerabilities

Exploited vulnerabilities

Detailed technical findings with proof-of-concept evidence and remediation steps. Click to expand.

Vulnerability 01
Kerberoastable Service Account with Domain Admin Privileges
Critical CVSS 9.8 +

The service account svc_veeam has a registered SPN, making it vulnerable to Kerberoasting. Any authenticated domain user can request a TGS ticket and crack the password offline. This account is a member of Domain Admins, meaning a successful crack grants full domain control. The password was last set on 2022-11-03.

Proof of Concept

$ impacket-GetUserSPNs evilcorp.local/jsmith -dc-ip 10.10.10.10 -request

ServicePrincipalName  : HTTP/VEEAM01.evilcorp.local
Account               : svc_veeam
MemberOf              : Domain Admins
PasswordLastSet       : 2022-11-03

TGS Hash: $krb5tgs$23$*svc_veeam$EVILCORP.LOCAL$...

Remediation

  • Remove svc_veeam from Domain Admins immediately
  • Implement Group Managed Service Accounts (gMSA)
  • Use 25+ character randomly generated passwords for traditional service accounts
  • Implement AES-256 encryption for Kerberos instead of RC4
  • Monitor for Kerberoasting via Event ID 4769 with RC4 encryption
Vulnerability 02
DCSync — Full Domain Credential Extraction
Critical CVSS 10.0 +

Using the compromised svc_veeam Domain Admin account, a DCSync attack was performed against DC01 to replicate all domain password hashes. This extracted NTLM hashes for all 847 domain accounts, including the built-in Administrator and krbtgt accounts. The krbtgt hash enables Golden Ticket attacks for persistent domain access.

Proof of Concept

$ impacket-secretsdump evilcorp.local/svc_veeam@10.10.10.10 -just-dc-ntlm

Administrator:500:...: 92937ae7...f3 :::
krbtgt:502:...: f3bc61e9...5f :::
[...] 847 accounts dumped

Remediation

  • Remove svc_veeam from Domain Admins immediately
  • Audit all accounts with DCSync rights (Replicating Directory Changes)
  • Rotate krbtgt password twice to invalidate Golden Tickets
  • Reset all user passwords following a breach
  • Implement tiered administration model to limit DA usage
Vulnerability 03
Ransomware Simulation via Malicious GPO Deployment
Critical CVSS 10.0 +

Using Domain Admin privileges, a GPO was created and linked to the domain root. This GPO deploys a startup script to all 312 domain-joined computers simulating a ransomware event. This demonstrates that an attacker who compromises a single Domain Admin service account can deploy ransomware enterprise-wide via native AD tooling, bypassing endpoint security.

Proof of Concept

GPO Name   : "YOURNETWORKISNOTASSECUREASYOUTHINK"
GPO GUID   : {8F4E2A1B-3C5D-4E6F-A7B8-9C0D1E2F3A4B}
Linked to  : DC=evilcorp,DC=local (domain root)
Affects    : 312 domain-joined computers

// Script: creates ransom note + event log entry
@echo off
echo YOUR NETWORK IS NOT AS SECURE AS YOU THINK > %USERPROFILE%\Desktop\README_RANSOM.txt
eventcreate /T WARNING /ID 666 /L APPLICATION /D "Ransomware simulation"

Remediation

  • Remove svc_veeam from Domain Admins
  • Restrict who can create/modify GPOs (audit GPO delegation)
  • Monitor for new GPO creation via Event IDs 5136/5137
  • Implement LAPS for local admin passwords
  • Deploy GPO change monitoring/alerting
  • Consider Privileged Access Workstations (PAWs)
Vulnerability 04
SMB Signing Not Required
High CVSS 7.4 +

23 hosts in the domain do not enforce SMB signing, making them vulnerable to NTLM relay attacks. An attacker who can perform a man-in-the-middle attack can relay NTLM authentication to these hosts to gain unauthorized access.

Proof of Concept

$ crackmapexec smb 10.10.10.0/24 --gen-relay-list

23 targets identified without SMB signing requirement.

Remediation

  • Enable SMB signing via Group Policy: Computer Configuration → Policies → Windows Settings → Security Settings → Local Policies → Security Options
  • Set "Microsoft network server: Digitally sign communications (always)" = Enabled

06 — Remediation

Recommended actions

Prioritized remediation steps to address the identified vulnerabilities and harden the evilcorp.local domain.

Immediate
Remove svc_veeam from Domain Admins
This is the root cause of the entire attack chain. Removing DA privileges eliminates the single-hop path to domain compromise.
Immediate
Rotate krbtgt password twice
The krbtgt hash was extracted via DCSync. A double rotation invalidates all existing Golden Tickets.
Immediate
Reset all domain passwords
All 847 account NTLM hashes were extracted. A domain-wide password reset is required to invalidate compromised credentials.
Short-Term
Implement gMSA for service accounts
Group Managed Service Accounts use auto-rotated 240-character passwords, eliminating Kerberoasting risk entirely.
Short-Term
Enforce SMB signing domain-wide
23 hosts currently lack SMB signing. Enable via Group Policy to prevent NTLM relay attacks.
Short-Term
Monitor Kerberoasting (Event ID 4769)
Alert on TGS requests with RC4 encryption type (0x17). This detects Kerberoasting attempts in real time.
Medium-Term
Implement tiered administration
Separate Tier 0 (DCs), Tier 1 (servers), and Tier 2 (workstations) administration to contain breaches.
Medium-Term
Audit and restrict GPO delegation
Monitor GPO creation/modification via Event IDs 5136/5137. Restrict who can link GPOs to the domain root.
